# Spec 001 — PyaServ MVP

## Pitch

Two-sided marketplace for services in Paraguay. **No platform payments, no commissions.**
- **Specialists** publish profiles + service listings.
- **Clients** publish work requests.
- Either side can start a private inquiry on a listing or a request.
- Reviews come later (out of MVP scope).

## Domain model

### `specialist_profiles`
A user becomes a specialist by filling this out once. One profile per user.

| field | type | notes |
| --- | --- | --- |
| `id` | TEXT PK | UUID v7 |
| `user_id` | TEXT UNIQUE | → `users(id)` |
| `display_name` | TEXT | "Diego Cabrera" |
| `headline` | TEXT | one-liner shown in cards |
| `bio` | TEXT | long description, markdown |
| `phone` | TEXT | revealed after inquiry is opened |
| `whatsapp` | TEXT NULLABLE | optional |
| `barrio` | TEXT | Asunción neighborhood, used for filtering |
| `lat` / `lng` | REAL NULLABLE | optional precise location |
| `photo` | TEXT NULLABLE | media key (R2 / KV fallback) |
| `verified` | INTEGER | 0 / 1; admin flips manually for now |
| `status` | TEXT | `active` / `paused` |
| `created_at` / `updated_at` | INTEGER | unix seconds |

### `listings`
A service a specialist offers. One specialist can post many.

| field | type | notes |
| --- | --- | --- |
| `id` | TEXT PK | UUID v7 |
| `specialist_id` | TEXT | → `specialist_profiles(id)` ON DELETE CASCADE |
| `category` | TEXT | enum below |
| `title` | TEXT | "Plomería 24h" |
| `description` | TEXT | markdown |
| `price_from_gs` | INTEGER NULLABLE | guaraníes; null = "ask" |
| `price_unit` | TEXT NULLABLE | `hour` / `job` / null |
| `photo` | TEXT NULLABLE | media key |
| `status` | TEXT | `active` / `paused` |
| `created_at` / `updated_at` | INTEGER |

Categories (hardcoded for MVP): `plumbing`, `electrical`, `cleaning`, `repair`, `beauty`, `teaching`, `photography`, `translation`, `events`, `other`.

### `requests`
A client posts a work request.

| field | type | notes |
| --- | --- | --- |
| `id` | TEXT PK |
| `client_id` | TEXT | → `users(id)` |
| `category` | TEXT | same enum as listings |
| `title` | TEXT | "Necesito plomero urgente" |
| `description` | TEXT |
| `budget_gs` | INTEGER NULLABLE |
| `barrio` | TEXT |
| `status` | TEXT | `open` / `closed` |
| `created_at` / `updated_at` | INTEGER |

### `inquiries`
A private chat between two users about either a listing or a request.

| field | type | notes |
| --- | --- | --- |
| `id` | TEXT PK |
| `subject_type` | TEXT | `listing` or `request` |
| `subject_id` | TEXT | the listing or request id |
| `client_user_id` | TEXT | → `users(id)` |
| `specialist_user_id` | TEXT | → `users(id)` |
| `last_message_at` | INTEGER | for sorting |
| `created_at` | INTEGER |

`UNIQUE(subject_type, subject_id, client_user_id, specialist_user_id)` — one thread per (subject, pair).

### `messages`

| field | type | notes |
| --- | --- | --- |
| `id` | TEXT PK |
| `inquiry_id` | TEXT | → `inquiries(id)` ON DELETE CASCADE |
| `sender_user_id` | TEXT | → `users(id)` |
| `body` | TEXT |
| `created_at` | INTEGER |

## API surface (v1)

| method | path | auth | purpose |
| --- | --- | --- | --- |
| GET | `/v1/specialists` | none | list, filter by category/barrio |
| GET | `/v1/specialists/:id` | none | detail + listings |
| POST | `/v1/specialists` | required | create profile |
| PATCH | `/v1/specialists/:id` | owner | edit |
| GET | `/v1/listings` | none | list by category/barrio |
| POST | `/v1/listings` | required | create listing for self |
| PATCH | `/v1/listings/:id` | owner | edit |
| DELETE | `/v1/listings/:id` | owner | delete |
| GET | `/v1/requests` | none | list, filter by category/barrio |
| GET | `/v1/requests/:id` | none | detail |
| POST | `/v1/requests` | required | client posts a request |
| PATCH | `/v1/requests/:id` | owner | edit / close |
| GET | `/v1/inquiries` | required | mine (both sides) |
| GET | `/v1/inquiries/:id` | participant | thread + messages |
| POST | `/v1/inquiries` | required | open thread on listing or request |
| POST | `/v1/inquiries/:id/messages` | participant | reply |

## Frontend pages

| route | purpose |
| --- | --- |
| `/` | landing → two big cards (already exists, mostly fine) |
| `/specialists/` | filterable list of profiles |
| `/specialists/[id]` | profile + their listings + "Contactar" CTA |
| `/clients/` | filterable list of requests (renamed for SEO: stays at this path) |
| `/clients/[id]` | request detail + "Postularme" CTA |
| `/me/` | dashboard (specialist profile editor, my listings, my requests, my inquiries) |
| `/me/inquiries/[id]` | one chat |

Anonymous browsing is allowed for listings/requests. Login is required to post anything or open an inquiry.

## Out of scope (MVP)

- Ratings + reviews
- Saved searches / notifications
- Payment escrow
- Multi-language (Spanish only for now; `@pya-company/i18n` is already in place for later)
- Admin panel (verification flag is set via D1 console for MVP)
